# Good First Issues to Create

Here is a list of issues you can copy and paste into your GitHub Issues tracker to get started. Label them as `good first issue` and `help wanted`.

---

## Issue 1: Fix Typo in README

**Title:** Fix typo in README.md

**Body:**
There is a small typo in the `README.md` file.
In the "Features" section, it says "Secure Storage - Encrypted credentials in NVS".
Please check if "NVS" should be expanded to "Non-Volatile Storage" for clarity.

**Labels:** documentation, good first issue

---

## Issue 2: Add Comments to BasicProvisioning Example

**Title:** Add comments to BasicProvisioning example

**Body:**
The example at `examples/BasicProvisioning/BasicProvisioning.ino` could use more comments explaining what each line does.
Please add comments to the `setup()` function to make it easier for beginners to understand.

**Labels:** documentation, good first issue

---

## Issue 3: Add Unit Test for WiFi Connection

**Title:** Add unit test for WiFi connection logic

**Body:**
We need more unit tests!
Please add a test case in `tests/test_wifi.cpp` to verify that the `connect()` function returns `false` if an empty SSID is provided.

**Labels:** testing, good first issue

## Issue 4: Create a "Hello World" Example

**Title:** Create a simple "Hello World" example

**Body:**
Create a new example sketch called `examples/HelloWorld/HelloWorld.ino`.
It should just initialize WiBLE and print "Hello World" to the Serial monitor when the device is provisioned.
